# 双击单元格开启全局编辑模式 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 让主表 `Text/Number` 单元格在双击时仅开启全局 `编辑` 模式，不自动激活当前 cell，也不影响现有单击激活、选区和非文本类单元格交互。

**Architecture:** 现有全局表级编辑状态真相在 `App.tsx` 的 `tableTextEditMode`，并通过 `tableSnapshot.textEditable` 下发到 `DataTable -> table-columns -> CellRenderer -> TextCellSurface`。本次实现保持这条链路不变，只新增一个 `enable-only` 回调自上而下透传到 `TextCellSurface` 的 display layer，再由 `onDoubleClick` 触发上行请求；`TableTextCellEditor`、`activeTextCellId` 和现有输入状态机不改。

**Tech Stack:** React 18, TypeScript, TanStack Table, Playwright e2e (`tests/data-editor.spec.ts`), Node source assertions (`tests/view-state.test.mjs`)

- [ ] **Step 1: 在 `App.tsx` 新增 enable-only 回调，而不是复用 toggle**

在 `const [tableTextEditMode, setTableTextEditMode] = useState(false);` 附近新增：

```ts
const enableTableTextEditMode = useCallback(() => {
  setTableTextEditMode((current) => current ? current : true);
}, []);
```

然后在传给 `DataTable` 的 props 上新增：

不要给 `DataTable` 再新增一条平行的 `onEnableTextEditMode` props。当前表格状态已经由 `tableSnapshot` 驱动，enable-only 回调应沿同一 `snapshot` 通道传递。

改为在 `tableSnapshot` 构造处新增：

```ts
const tableSnapshot = useMemo<TableSnapshot>(() => ({
  // ...
  textEditable: tableTextEditMode,
  onEnableTextEditMode: enableTableTextEditMode,
  onRegisterActiveTextEditor: registerActiveTextEditor,
}), [
  // ...
  tableTextEditMode,
  enableTableTextEditMode,
  registerActiveTextEditor,
]);
```

不要改现有：

```ts
onToggleTableTextEditMode={() => setTableTextEditMode((value) => !value)}
```

因为按钮仍然需要 toggle，而双击路径只允许 enable-only。

- [ ] **Step 2: 给 `DataTable` / `TableSnapshot` / runtime 增加 enable-only 通道**

在 `src/table/DataTable.tsx` 中新增：

```ts
export type TableSnapshot = {
  // ...
  textEditable: boolean;
  onEnableTextEditMode?: () => void;
  onRegisterActiveTextEditor?: ActiveTextEditorRegistrar;
};
```

并在 `tableColumnsRuntime` 中透传：

```ts
onEnableTextEditMode: snapshot.onEnableTextEditMode ?? (() => {}),
```

保持 `activeTextCellId`、`handleActivateTextCell`、`handleDeactivateTextCell` 不变。

- [ ] **Step 3: 在 `table-columns.tsx` 和 `CellRenderer.tsx` 中只对 Text/Number 装配回调**

在 `src/table/table-columns.tsx` 的 runtime 类型里新增：

```ts
onEnableTextEditMode: () => void;
```

并在 `TableColumnCellView` 里传给 `CellRenderer`：

```tsx
onEnableTextEditMode={runtime.onEnableTextEditMode}
```

在 `src/table/CellRenderer.tsx` 的 props 增加：

```ts
onEnableTextEditMode?: () => void;
```

然后只在 `Text` / `Number` 分支传给 `TextCellSurface`：

```tsx
<TextCellSurface
  // ...
  onEnableEditing={onEnableTextEditMode}
/>
```

不要在 `Select` / `Multi-select` / `Relation` / `Document` 分支新增任何双击逻辑。

- [ ] **Step 4: 在 `TextCellSurface.tsx` 的 display layer 上接入 `onDoubleClick`**

把 props 改成：

```ts
type TextCellSurfaceProps = {
  // ...
  onEnableEditing?: () => void;
};
```

新增处理函数：

```ts
function handleRequestEnableEditing(event: ReactMouseEvent<HTMLDivElement>) {
  event.stopPropagation();
  if (editable) return;
  onEnableEditing?.();
}
```

并仅在 display layer 上接入：

```tsx
<div
  className={`table-text-cell-display text-cell-display-layer editable-cell cell-display cell-text-content ${wrapped ? "cell-text-wrap" : ""}`}
  data-cell-role="content"
  data-display-type={displayType.toLowerCase()}
  data-wrap-mode={wrapped ? "wrap" : "truncate"}
  data-zero-value={isZeroValue ? "true" : "false"}
  tabIndex={editable && !active ? 0 : undefined}
  onClick={handleActivate}
  onDoubleClick={handleRequestEnableEditing}
  onKeyDown={handleActivateByKeyboard}
>
```

这一步的关键是：

- 双击只在 `editable === false` 时生效
- 双击后不调用 `onActivate`
- 必须显式观察并验证：同一双击手势的第二个 click 不会因为 enable-only 状态切换把当前 cell 推进 `editable-active`
- 不给 overlay、`td`、`TableCellFrame` 增加双击
