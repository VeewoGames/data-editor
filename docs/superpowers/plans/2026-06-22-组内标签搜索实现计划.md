# 组内标签搜索 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 为共享视图第二层组内标签增加局部搜索入口，只过滤当前展开组的标签显示，不影响 active view 的 `query`。

**Architecture:** 在 `ViewTabs` 内增加组内标签过滤本地状态，并把第二层结构拆成搜索区、tabs 区和组内创建区三个并列区域。复用 `ExpandableSearch` 的展开交互，但通过独立 class 和局部 ref 隔离现有 `.search-box` 语义，同时补 Playwright 回归覆盖。

**Tech Stack:** React 18, TypeScript, Playwright, Node test runner

### Task 2: 实现第二层搜索结构和行为

**Files:**
- Modify: `C:\Code\data-editor\src\components\ViewTabs.tsx`
- Modify: `C:\Code\data-editor\src\components\ExpandableSearch.tsx`
- Modify: `C:\Code\data-editor\src\styles.css`

- [ ] **Step 1: 扩展可复用搜索控件的最小接口**

为 `ExpandableSearch` 增加可选的：

- `inputRef`
- `inputClassName`
- `onEscape`
- `iconAriaLabel`

保持现有 toolbar 行为不变。

- [ ] **Step 2: 在 `ViewTabs` 内增加本地过滤状态和重置逻辑**

增加 `groupTabFilter`，在 `expandedGroupId` 变化时清空，并基于 `expandedGroup.views` 计算过滤后的列表，确保 active 组内 tab 始终保留。

- [ ] **Step 3: 拆分第二层结构**

把第二层从单一 `tablist` 改成：

- 外层 `view-tabs-group-row`
- 左侧 `group-tab-search`
- 中间 `view-tabs-group-tabs` 作为真实 `tablist`
- 右侧组内 `+`

- [ ] **Step 4: 接好焦点和 Escape 规则**

通过局部 ref 聚焦组内搜索，不改现有 `.search-box` 查询行为；实现空值 `Escape` 收起、非空 `Escape` 清空。

- [ ] **Step 5: 调整样式**

补齐第二层搜索布局、宽度、最小尺寸和 tabs 区 `min-width: 0`，保证横向溢出和现有视觉风格一致。
