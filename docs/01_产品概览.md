# Data Editor 产品概览

## 概述

`data-editor` 是一个本地 Web 数据编辑器，用来查看和修改项目中的 JSON / CSV 数据文件。它的目标不是替代源码编辑器，也不是通用数据库产品，而是给项目型数据提供一个更接近表格数据库的编辑体验，同时保留本地文件、Git diff 和项目结构的可控性。

## 产品定位

- 运行形态：本地 HTTP 服务 + 浏览器前端。
- 主要用户：项目开发者、数据设计人员、需要快速检查或批量维护配置数据的 AI agent。
- 核心场景：浏览数据文件、切换集合、编辑行列、查看嵌套结构、维护 relation / backlink / primary key，并通过自动保存写回源文件。

## 当前已经具备的能力

- 通过本机 registry 管理多个项目，并在 UI 中切换 active project。
- 为项目声明多个数据源，并用虚拟路径浏览白名单数据文件。
- 加载 JSON 和 CSV，并构建统一的 `DocumentModel`。
- 在主表中编辑顶层字段，并用主表文本编辑模式保护普通文本单元格输入。
- 在详情面板中编辑嵌套对象和数组。
- 为字段指定显示类型。
- 为字段配置 relation。
- 从 relation 自动派生 backlink 展示列和详情面板引用区。
- 为集合维护 primary key，并分析主键改名影响。
- 为 collection 提供团队共享视图、搜索、筛选、排序和个人 draft。
- 创建共享视图副本，并复制当前有效视图快照与当前用户的个人布局。
- 持久化个人文件树展开/排序、视图级列布局、主题和基础字号。
- 刷新后恢复当前文件、collection、active view 和表格滚动位置。
- 在文件移动或重命名后迁移相关视图与页面上下文配置。
- 自动保存业务数据、项目级视图配置和个人视图配置。
- 服务断开后通过 recovery bridge 尝试恢复。

## 产品不做什么

- 不替代 Notion、doc_sync 或 notion_sync。
- 不负责远程协作或多人同时编辑。
- 不把业务数据迁移成新的持久化格式。
- 不在保存时自动“修复”所有不合法数据。
- 不把历史方案文档继续当作正式产品说明的一部分。

## 当前运行模型

- 前端：React + TypeScript + Vite。
- 本地 API：Node.js HTTP 服务。
- 默认主服务端口：`8787`。
- 默认 recovery bridge 端口：`8791`。
- 项目级配置：`.data-editor/view-config.json` 和 `.data-editor/shared-views.json`。
- 个人视图配置：通过 `/api/view-profile` 读写 `.data-editor/view-configs/<profile>.json`。
- 本地页面上下文：`localStorage["data-editor:page-context"]`。
- 本地主题/字号 fallback：`localStorage["data-editor:ui-theme"]` 和 `localStorage["data-editor:ui-font-size"]`。

## 建议理解方式

理解这个产品时，先把它看成三层：

1. 文件和集合层：决定“数据从哪里来”。
2. 视图和字段层：决定“怎么展示和编辑”。
3. 关系和保护层：决定“哪些字段有结构语义，哪些操作需要更强约束”。
