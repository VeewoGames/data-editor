# Data Editor 系统结构

## 概述

这一节从代码组织角度描述系统结构，方便团队成员和 AI agent 快速定位入口。

## 顶层运行结构

- `open.mjs`
  - 推荐启动入口
  - 确保 recovery bridge 存在
  - 通过 controller 启动主服务
- `server.mjs`
  - 本地 HTTP 服务
  - 提供项目 registry、文件、文档、配置、共享视图、保存、构建、关闭等 API
- `stop.mjs`
  - 按 registry home 下的运行时状态安全停止主服务和 recovery bridge

## 前端入口

- `src/main.tsx`
  - 应用挂载入口
- `src/App.tsx`
  - 顶层容器
  - 串联项目切换、文件加载、视图配置、relation/backlink、自动保存、命令保存、恢复和对话框状态

`App.tsx` 当前是系统级编排中心。

## 文本编辑与保存状态模块

- `src/editing/useStableDraftInput.ts`
  - 为文本输入维护本地 draft、输入法组合态和提交节奏，避免后台自动保存导致 React 受控值重放后打断输入。
- `src/editing/StableTextInput.tsx`
  - 提供单行和多行稳定文本输入组件，供详情页复用。
- `src/editing/TableTextCellEditor.tsx`
  - 主表普通文本单元格编辑器，只在 `ViewTabs` 的 `编辑` 模式开启时启用。
- `src/editing/types.ts`
  - 定义当前活动文本编辑器的 `flushDraft()` 契约，供文件切换、profile 切换、刷新构建、关闭服务和页面卸载前统一 flush。

保存状态分为两层：

- `autosaveState`
  - 表示后台自动保存的展示状态。普通业务数据 autosave 不再驱动全局禁用态，也不应打断当前文本输入。
- `commandSaving`
  - 表示显式命令保存或结构性写配置流程，例如共享视图发布、primary key 同步确认、刷新构建和关闭服务前 flush。该状态仍可禁用会造成并发结构变更的按钮。

## 主表 Cell Layout Contract

主表显示契约当前以 `DataTable.tsx` 为入口，核心约定如下：

- 数据内容格 `td` 显式带 `data-cell-kind="data"`、`data-column-field` 和 `data-wrap-mode="truncate|wrap"`。
- 工具格与尾部占位格不复用数据内容格契约，分别使用 `data-cell-kind="row-action"` 和 `data-cell-kind="add-column-spacer"`。
- 普通文本和标题文本通过 `data-cell-role="content" | "title" | "title-text"` 暴露稳定 DOM 合同，供样式和 e2e 断言使用。
- token / relation / select 类 trigger 通过 `surface="table" | "detail"` 区分主表与 detail panel；主表布局不能顺手泄漏到 detail panel。
- 普通文本单元格只有在主表 `textEditable` 为 true 时才渲染 `TableTextCellEditor`；Select、Multi-select、Relation 分支不受 `textEditable` 影响。
- 标题列继续保留打开详情的主路径，不作为主表文本编辑入口。

当前 `wrap` 语义已经拆成两层：

- `FieldConfig.wrapped` 仍是字段级内容展示偏好。
- `hasWrappedField` 仍是当前实现中切换主表自然行高 / 非虚拟滚动的触发条件。

两者目前共享同一来源，但语义上不再视为同一概念；动态行高虚拟化仍不在本轮处理范围。

## API 与持久化相关模块

- `src/api/client.ts`
  - 前端 API client
- `src/project-registry.mjs`
  - 本机项目清单读写、校验、默认项目注册和 active project 持久化
- `src/project-context.mjs`
  - 根据项目定义生成运行期上下文，包括项目根目录、数据源、配置路径和运行目录
- `src/file-service.mjs`
  - 多数据源文件枚举、虚拟路径解析、边界检查和直接写入
- `src/view-config.mjs`
  - 项目级视图配置读写
- `src/shared-views.mjs`
  - 团队共享视图配置读写，目标文件为项目根目录下的 `.data-editor/shared-views.json`
- `src/view-profile.mjs`
  - 个人视图配置读写，包括 profile 模式下的共享视图 draft 和按 `collection + viewId` 保存的个人列布局
- `src/view-state-storage.mjs`
  - 本地 localStorage 视图状态、文件排序和共享视图 draft 读写；负责按 `collection + viewId` 读写和重置个人列布局
- `src/save-documents.mjs`
  - 多文档保存流程

## 共享视图与筛选排序模块

- `src/view/shared-view-normalize.mjs`
  - 规范化 `SharedViewsConfig`、`CollectionView`、`FilterGroup`、`SortRule` 和个人 draft 状态；过滤无效字段、无效 operator 和无效排序方向。
- `src/view/view-state.mjs`
  - 解析 collection 的共享视图列表、默认视图和 active view；合并共享视图与个人 draft；处理新建、重命名、删除、排序 draft、重置和保存 draft 到共享配置。
- `src/view/filtering.mjs`
  - 对主表 rows 应用当前 view 的 `query` 和单层 `and` 筛选规则，并保留原始行索引用于回写。
- `src/view/sorting.mjs`
  - 对筛选后的 rows 应用 `SortRule[]`，支持数字和字符串比较，并提供列头排序更新逻辑。

列隐藏、换行、列宽、列顺序和详情字段顺序当前仍由 `src/view-profile.mjs` 与浏览器本地 view layout state 承载，属于个人布局偏好，不属于共享视图发布链路。

## 运行时状态与恢复

- `src/runtime-state.mjs`
  - 运行时状态文件读写
- `recovery-bridge.mjs`
  - 主服务恢复与 controller 逻辑的宿主

多项目模式下，运行时状态、controller 状态和日志以 registry home 为根，不再绑定某一个 active project。默认 registry home 为 `%APPDATA%\data-editor`，可以通过 `DATA_EDITOR_HOME` 或 `--registry-home` 覆盖。

业务配置仍以项目根目录为根：

```text
<project>/.data-editor/view-config.json
<project>/.data-editor/shared-views.json
<project>/.data-editor/view-configs/
```

数据文件默认通过 direct-write 保存，不再维护项目级 `.data-editor/backups/` 目录。

## 面向 agent 的快速定位建议

如果问题涉及：

- 项目切换/registry：先看 `server.mjs`、`project-registry.mjs`、`App.tsx`
- 文件加载/保存：先看 `server.mjs`、`file-service.mjs`、`project-context.mjs`
- relation/backlink：先看 `fieldRole.ts`、`backlinkGrid.ts`、`relationMaintenance.ts`
- 主表行为：先看 `DataTable.tsx`
- 文本输入稳定性：先看 `src/editing/useStableDraftInput.ts`、`StableTextInput.tsx`、`TableTextCellEditor.tsx`
- 共享 view / filter / sort：先看 `App.tsx`、`ViewTabs.tsx`、`ViewFilterBar.tsx`、`shared-views.mjs`、`view-state.mjs`、`shared-view-normalize.mjs`、`filtering.mjs`、`sorting.mjs`
- 详情编辑：先看 `DetailPanel.tsx`
- 服务重启/停止：先看 `open.mjs`、`recovery-bridge.mjs`、`service-lifecycle.mjs`、`stop.mjs`、`runtime-state.mjs`
- 视图配置：先看 `viewConfig.ts`、`view-config.mjs`、`shared-views.mjs`、`view-profile.mjs`

## 服务收尾结构

- `src/service-finalizer.mjs`
  - 任务收尾时的服务可用性检查、保护 PID 计算、临时资源清理计划和恢复决策。
- `scripts/service-finalize.mjs`
  - 命令行入口，提供 `npm run service:status` 和 `npm run service:finalize`。
- `package.json`
  - `service:status` 只查看状态，不执行清理或恢复。
  - `service:finalize` 执行服务健康检查、必要恢复和安全临时资源清理。
