# Data Editor 信息架构与交互

## 概述

`data-editor` 的界面围绕“选文件、选集合、看主表、改详情、保存结果”这个主工作流组织。

## 主界面分区

### Sidebar

`Sidebar` 负责三个任务：

- 选择文件
- 选择 collection
- 展示 metadata 和主键候选提示

它是用户进入具体数据上下文的入口。

### Toolbar

`Toolbar` 负责全局动作：

- 搜索
- 选择视图配置
- 保存
- 重置视图
- 刷新构建
- 关闭服务
- 管理隐藏字段

这里承载的是“当前上下文的全局操作”，不是行级编辑。

### DataTable

`DataTable` 是主表区域，负责：

- 展示顶层字段
- 行内编辑
- 排序
- 列宽调整
- 列顺序拖拽
- 字段隐藏
- 新增行、删行
- 打开详情

主表优先承担高频、低上下文切换的编辑操作。

### DetailPanel

`DetailPanel` 负责：

- 查看和编辑当前记录的完整字段
- 编辑嵌套对象和数组
- 展示 relation backlink
- 展示 primary key 改名后的影响与同步入口

它是主表之外的深度编辑区。

### RelationConfigDialog

`RelationConfigDialog` 用于把某个字段显式配置为 relation。配置项包括：

- 目标文件
- 目标集合
- 目标主键
- 单值 / 多值模式
- 标题字段优先级
- 是否允许缺失引用

## 主工作流

### 1. 打开文件

用户先在 `Sidebar` 中选择文件，然后选择具体 collection。

### 2. 浏览和筛选

用户在 `Toolbar` 中搜索，在 `DataTable` 中排序、查看、调整视图。

### 3. 编辑数据

- 简单字段：直接在主表或详情中编辑
- 嵌套字段：进入详情面板继续编辑
- relation 字段：使用 relation editor 选择目标记录

### 4. 处理结构语义

如果字段有 special role：

- relation：显示为引用选择器
- backlink：只读展示引用来源
- primary key：受额外保护

### 5. 保存

保存时系统分别处理：

- 业务数据
- 项目级配置
- 个人视图配置

## 交互约束

- 主表只承载顶层、适合高频查看的内容。
- 嵌套结构不在主表内强行铺平。
- relation 和 backlink 有独立视觉语义，不等同于普通 select。
- 主键字段不应被当作普通 relation 字段处理。
- 详情面板是深度编辑器，不是简单信息预览层。
