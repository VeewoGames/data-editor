# Data Editor 信息架构与交互

## 概述

`data-editor` 的界面围绕“选文件、选集合、看主表、改详情、自动保存结果”这个主工作流组织。

## 主界面分区

### Sidebar

`Sidebar` 负责三个任务：

- 在文件树中选择文件
- 选择 collection
- 展示 metadata 和主键候选提示

它是用户进入具体数据上下文的入口。

当前 Sidebar 已经是树形导航，而不是单纯平铺文件列表：

- 支持 source / folder / file 层级展示
- 支持文件夹展开与折叠
- 支持同父节点内的拖拽排序
- 个人排序和展开状态可持久化到 profile 或浏览器本地状态

### Toolbar

`Toolbar` 负责全局动作：

- 搜索
- 选择个人视图配置 profile
- 展示自动保存状态
- 切换主题和基础字号
- 重置视图
- 刷新构建
- 关闭服务
- 管理隐藏字段

这里承载的是“当前上下文的全局操作”，不是行级编辑。

### ViewTabs

`ViewTabs` 位于主表上方，负责当前 collection 的共享视图切换、视图 tab 管理，以及当前视图的轻量入口：

- 顶层第一行：同时展示顶层普通视图和视图组。
- 点击视图组：激活该组，并在第二行展开该组内的具体视图标签。
- 顶层 `+`：只负责创建顶层普通视图或顶层视图组。
- 组内第二行 `+`：只在当前展开组内出现，只负责向该组内创建子视图。
- 视图组菜单：支持重命名组、组内创建视图、删除组。
- 新建视图：基于当前 active view 快照创建新的共享视图；如果当前上下文在某个组内，可通过组内 `+` 直接创建到该组中。
- 创建视图副本：复制来源视图的当前有效快照；如果当前用户已有该视图的个人布局，也一起复制到新 view。
- 重命名、删除：直接修改共享视图配置。
- 拖拽排序：支持顶层排序、组内排序、把顶层视图拖入组，以及把组内视图拖回顶层；结构变化先进入个人 draft。
- 当前 active tab：点击时打开 Notion-like 浮框；点击其他 tab 时只切换视图，不额外打开浮框。
- `筛选`：位于当前视图搜索图标左侧，用于显示或隐藏筛选栏；当前视图已有筛选时文字变蓝。
- `编辑`：位于 `筛选` 和 `调整` 之间，用于开启或关闭主表普通文本单元格编辑模式；关闭时普通文本单元格只负责展示和打开详情。
- 展开式搜索：平时只显示放大镜图标，点击后展开并写入当前 active view 的 `query`。

视图组本身只是导航容器，不承载筛选、排序或查询语义；真正的 active 对象始终是具体 view。视图 tab 上的未保存标记表示该视图有个人 draft 尚未保存到共享配置。

### ViewFilterBar

`ViewFilterBar` 位于 view tabs 和主表之间，可由 `ViewTabs` 中的 `筛选` 按钮隐藏或显示。它负责当前 active view 的筛选、排序和共享发布入口：

- `排序`：位于筛选栏左侧，打开排序 popover。
- `+ 筛选`：打开字段选择菜单，选择字段后为当前视图新增对应类型的默认筛选规则。
- 顶部普通 filter chips：展示当前视图仍停留在顶层的筛选规则，点击 chip 打开对应字段类型的编辑 popover；这些 chips 都是真实生效条件，且彼此固定为 `AND`。
- 高级筛选入口 chip：当当前视图存在 `advancedRoot` 时显示；顶部只能有一个，点击后打开高级筛选面板。被合并进高级树的规则不会继续以普通 chip 形式显示在顶部。
- 排序 chips：展示当前视图已有排序规则，点击 chip 打开同一个排序 popover。
- `重置`：位于筛选栏右侧；仅在当前 active view 或 view tab 顺序存在个人 draft 时显示，点击后丢弃对应 draft。
- `为所有人保存`：位于筛选栏右侧；仅在存在可发布 draft 时显示，点击后把当前 active view 的个人 draft 与当前 collection 的结构 draft 一起覆盖保存为团队共享配置。

当前筛选采用“双层组合”模型：

- 顶部普通 chips 固定 `AND`
- 唯一高级筛选入口 chip 代表一棵真实生效的高级筛选树
- 总体结果是“所有顶部普通规则成立”且“高级筛选树成立”
- 高级筛选树支持 `AND/OR` 分组与最多 3 层嵌套，其中 `advancedRoot` 记作第 1 层
- 第一版不支持把高级树内的某条 rule 直接移回顶部普通 chips

### DataTable

`DataTable` 是主表区域，负责：

- 展示顶层字段
- 行内编辑：普通文本字段受 `编辑` 模式控制；Select、Multi-select、Relation 不受该模式影响，仍可在主表内操作
- 排序
- 列宽调整
- 列顺序拖拽
- 字段隐藏
- 新增行、删行
- 打开详情

主表优先承担高频、低上下文切换的编辑操作。标题列仍优先承担打开详情的导航职责，不进入主表文本编辑模式。

### DetailPanel

`DetailPanel` 负责：

- 查看和编辑当前记录的完整字段
- 编辑嵌套对象和数组
- 展示 relation backlink
- 展示 primary key 改名后的影响与同步入口

它是主表之外的深度编辑区。

### RelationConfigDialog

`RelationConfigDialog` 用于把某个字段显式配置为 relation。配置项包括：

- 目标文件
- 目标集合
- 目标主键
- 单值 / 多值模式
- 标题字段优先级
- 是否允许缺失引用

## 主工作流

### 1. 打开文件

用户先在 `Sidebar` 文件树中选择文件，然后选择具体 collection。文件树的展开状态、同级排序和默认回退打开路径都属于个人导航偏好。

### 2. 浏览和筛选

用户先通过 `ViewTabs` 选择当前 collection 的 active view。当前视图搜索在 view tab 行通过展开式搜索完成；筛选栏可由 `筛选` 按钮隐藏或显示。用户通过 `ViewFilterBar` 添加顶部普通 chips、把需要组合的规则合并进唯一高级筛选入口 chip、编辑排序 chips，最后在 `DataTable` 中查看、编辑、调整列布局。

当前视图内的搜索、筛选和排序会先写入个人 draft；顶层/组内结构拖拽会先写入当前 collection 的结构 draft。用户可以继续切换视图或文件；draft 属于个人，不会立刻影响团队共享配置。筛选栏右侧的 `重置` 会丢弃对应 draft；`为所有人保存` 会把当前 active view 的 draft 和当前 collection 的结构 draft 直接覆盖保存为团队共享视图配置，其他用户下次加载共享视图时看到新的结果。

在视图分组模式下，顶层/组内结构变化不会立即写入共享配置，而是先进入当前 collection 的 `structureDrafts`。点击 `为所有人保存` 时，会和当前 active view 的内容 draft 一起写入团队共享配置；点击 `重置` 时，会一起丢弃当前 active view draft 和当前 collection 的结构 draft。

列隐藏、换行、列顺序、详情字段顺序和列宽仍属于个人布局偏好，由 profile 或浏览器本地状态按当前 active view 独立保存，不随 `为所有人保存` 发布到团队共享视图。

如果用户选择“创建视图副本”，系统会把当前看到的有效视图状态作为来源快照，而不是只复制上一次共享保存的静态配置。

### 3. 编辑数据

- 简单字段：在详情中直接编辑；在主表中需要先开启 `编辑` 模式后才能编辑普通文本字段
- 嵌套字段：进入详情面板继续编辑
- relation 字段：使用 relation editor 选择目标记录

### 4. 处理结构语义

如果字段有 special role：

- relation：显示为引用选择器
- backlink：只读展示引用来源
- primary key：受额外保护

### 5. 自动保存

业务数据默认自动保存，不再要求用户手动点击右上角保存。系统分别处理：

- 业务数据
- 项目级配置
- 个人视图配置
- 团队共享视图配置

详情面板和主表普通文本编辑都先写入本地稳定 draft，再按输入节奏提交到 document model。自动保存只负责后台持久化，不应打断当前输入焦点、输入法组合态或光标位置。切换文件、切换 profile、刷新构建、关闭服务和页面卸载前，会先 flush 当前活动文本编辑器的 draft，再等待 autosave 队列落盘。

刷新页面后，系统会优先恢复当前项目下最近一次打开的文件、collection、active view 和对应表格滚动位置；这些恢复状态只保存在浏览器本地页面上下文中。

如果当前 active view 位于某个视图组内，页面上下文还会恢复：

- 当前展开组 `expandedGroupId`
- 每个组上次活跃的子视图 `lastActiveViewIdByGroupId`

因此刷新后会先恢复组展开，再恢复组内 active view；如果该子视图已经失效，则回退到该组第一个子视图。

## 交互约束

- 主表只承载顶层、适合高频查看的内容。
- 嵌套结构不在主表内强行铺平。
- 主表 `编辑` 模式只控制普通文本字段；Select、Multi-select、Relation 的主表操作保持原有即时交互。
- relation 和 backlink 有独立视觉语义，不等同于普通 select。
- 主键字段不应被当作普通 relation 字段处理。
- 详情面板是深度编辑器，不是简单信息预览层。
- 共享视图保存是直接覆盖团队配置，不做冲突检测。
