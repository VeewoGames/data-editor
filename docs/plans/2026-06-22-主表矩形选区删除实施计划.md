# 主表矩形选区删除 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 在 `data-editor` 主表中实现鼠标拖拽矩形选区，并支持对 `Text`、`Number`、`Checkbox`、`Select`、`Multi-select` 按 `Delete` 批量清空，且不破坏现有单击交互。

**Architecture:** 选区状态集中放在 `DataTable`，单元格 renderer 只提供稳定元信息并消费选中态。普通值类型继续走 `onEditCell`，`Select` / `Multi-select` 通过独立的 option field 清空适配层翻译成现有 draft commit patch，再走 `onCommitSelectDraft` / `onCommitMultiSelectDraft`。

**Tech Stack:** React 18, TypeScript/TSX, TanStack Table, Radix Popover, Node `--test`, Playwright

---

## 方案概述

### 1. 总体目标和范围

本实施计划用于把 [2026-06-22-主表矩形选区删除设计.md](C:/Code/data-editor/docs/superpowers/specs/2026-06-22-主表矩形选区删除设计.md) 落成可执行代码。目标是让主表 `tbody` 里的当前可见数据格支持鼠标拖拽建立矩形选区，并用 `Delete` 一次性清空选区内容。

本次实施范围包含：

- 在 `DataTable` 增加矩形选区运行时状态。
- 给主表数据格补稳定 DOM 元信息与选中态样式。
- 实现主表层 `Delete` 批量清空。
- 增加 `Select` / `Multi-select` 的 option field 清空适配层。
- 增加表格层可消费的活跃 option field popover 状态。
- 补纯逻辑测试与 Playwright 回归。

本次不包含：

- 复制、粘贴、剪切。
- 键盘扩选、`Shift + 点击`。
- 跨边缘自动滚动扩选。
- detail panel 侧的矩形选区。
- `Relation`、`Document`、backlink、nested summary 的批量清空。

### 2. 各阶段任务概要

阶段一：先抽纯逻辑 helper。  
主要工作是把矩形范围命中、清空值映射、option field 清空 patch 生成做成纯函数；预期成果是关键合同先脱离 UI 固化下来；执行顺序先写 node 测试再实现。

阶段二：先补测试基座。  
主要工作是新增一个单文件覆盖五类字段的 scratch fixture，并补通用 `dragBetweenCells` helper；预期成果是后续拖拽和批量清空 e2e 都建立在同一套稳定基座上；执行顺序先补 fixture，再补 helper 并验证可调用。

阶段三：接入主表运行时状态与 DOM 合同。  
主要工作是改 `DataTable`、`table-columns` 和 `CellRenderer`，让每个数据格都有稳定坐标、选中态和拖拽入口；预期成果是主表能稳定显示矩形选区；执行顺序先写失败 e2e，再补最小实现。

阶段四：接入删除语义与 option field 关闭/清空。  
主要工作是给 `Delete` 增加批量调度，先关闭活跃的 `Select` / `Multi-select` popover，再执行普通值写回或 option patch commit；预期成果是五种支持类型都能批量清空；执行顺序先锁定 helper，再接主表。

阶段五：收敛交互冲突与总回归。  
主要工作是明确拖拽阻断单击、title 单击保留单格选区、非拖拽单击不回归，并补 Playwright 回归；预期成果是拖拽与原交互共存；执行顺序先补断言，再跑 `test` / `typecheck` / `build`。

### 3. 整体结构框架

实现结构分三层：

1. 纯逻辑层：`src/table/table-selection.mjs`
2. 表格运行时层：`src/table/DataTable.tsx`
3. 渲染/编辑器配合层：`src/table/table-columns.tsx`、`src/table/CellRenderer.tsx`、`src/table/OptionFieldEditor.tsx`

两个强约束必须写死：

- 选区状态只能由 `DataTable` 管理，不下沉到各 cell editor。
- `Select` / `Multi-select` 的批量清空只能通过适配层生成 commit patch，不能让 `DataTable` 直接拼 `OptionFieldEditor` 的内部草稿状态。

async function dragBetweenCells(page: Page, source: Locator, target: Locator) {
  const sourceBox = await source.boundingBox();
  const targetBox = await target.boundingBox();
  if (!sourceBox || !targetBox) throw new Error("dragBetweenCells requires visible source and target cells");
  await page.mouse.move(sourceBox.x + sourceBox.width / 2, sourceBox.y + sourceBox.height / 2);
  await page.mouse.down();
  await page.mouse.move(targetBox.x + targetBox.width / 2, targetBox.y + targetBox.height / 2, { steps: 8 });
  await page.mouse.up();
}
```

export function buildSelectionRect(anchor, focus) {
  return {
    rowStart: Math.min(anchor.visibleRowIndex, focus.visibleRowIndex),
    rowEnd: Math.max(anchor.visibleRowIndex, focus.visibleRowIndex),
    columnStart: Math.min(anchor.visibleColumnIndex, focus.visibleColumnIndex),
    columnEnd: Math.max(anchor.visibleColumnIndex, focus.visibleColumnIndex),
  };
}

export function isCellInsideRect(rect, coord) {
  return coord.visibleRowIndex >= rect.rowStart &&
    coord.visibleRowIndex <= rect.rowEnd &&
    coord.visibleColumnIndex >= rect.columnStart &&
    coord.visibleColumnIndex <= rect.columnEnd;
}

export function resolveClearValueByDisplayType(displayType) {
  if (displayType === "Text" || displayType === "Number") return "";
  if (displayType === "Checkbox") return false;
  if (displayType === "Select") return null;
  if (displayType === "Multi-select") return [];
  return undefined;
}

export function buildOptionFieldClearPatch({ mode, options, selectedValues }) {
  return {
    createdOptionValues: [],
    deletedOptionValues: [],
    nextOptionOrder: options.map((option) => option.value),
    nextOptions: options,
    nextSelectedValues: [],
    optionsChanged: false,
    orderChanged: false,
    renamedOptions: [],
    valueChanged: selectedValues.length > 0 || mode === "single",
  };
}
```

const [activeOptionFieldCellId, setActiveOptionFieldCellId] = useState<string | null>(null);
const optionFieldClosersRef = useRef<Record<string, () => void>>({});

function handleOptionFieldOpenState(cellId: string, open: boolean, close: () => void) {
  if (open) {
    optionFieldClosersRef.current[cellId] = close;
    setActiveOptionFieldCellId(cellId);
    return;
  }
  delete optionFieldClosersRef.current[cellId];
  setActiveOptionFieldCellId((current) => current === cellId ? null : current);
}

function closeActiveOptionField() {
  if (!activeOptionFieldCellId) return;
  optionFieldClosersRef.current[activeOptionFieldCellId]?.();
}

function clearSelectedCells() {
  if (!selectionRect) return;
  closeActiveOptionField();
  for (const cell of selectedCells) {
    const nextValue = resolveClearValueByDisplayType(cell.displayType);
    if (cell.displayType === "Select") {
      runtimeActionRef.current.onCommitSelectDraft(
        cell.rowIndex,
        cell.rowId,
        cell.fieldName,
        buildOptionFieldClearPatch({ mode: "single", options: cell.options, selectedValues: cell.selectedValues }),
      );
      continue;
    }
    if (cell.displayType === "Multi-select") {
      runtimeActionRef.current.onCommitMultiSelectDraft(
        cell.rowIndex,
        cell.rowId,
        cell.fieldName,
        buildOptionFieldClearPatch({ mode: "multi", options: cell.options, selectedValues: cell.selectedValues }),
      );
      continue;
    }
    if (nextValue !== undefined) {
      runtimeActionRef.current.onEditCell(cell.rowIndex, cell.rowId, cell.fieldName, nextValue);
    }
  }
}

### 1. 方案覆盖检查

- 测试基座（专用 fixture + 拖拽 helper）：Task 0 覆盖
- 鼠标拖拽建立矩形选区：Task 1、Task 2 覆盖

- `Select` / `Multi-select` 先关闭 popover 再删除：Task 3、Task 4 覆盖
- title 单击保留单格选区并打开 detail：Task 4 覆盖
- 拖拽阻断单击交互：Task 5 覆盖
- render-time data attributes 替代额外 DOM patch：Task 2、Task 5 覆盖

### 2. 关键实现边界

- `DataTable` 只能调度选区与写回，不直接理解 `OptionFieldEditor` 的复杂 UI 草稿。
- `Select` / `Multi-select` 的清空 patch 只能从 helper 生成，避免逻辑散落在 `DataTable`。
- `data-view-row-index` 和 `visibleRowIndex` 都必须代表过滤/排序后的整张视图索引，不能再拿 `originalRowIndex` 顶替。
- title cell 的“单格选区 + 打开 detail”是设计显式要求，不能为了省事把 title 排除出单击选区模型。
- `surface === "table"` 是主表 option field 活跃态注册的硬边界，detail panel 不允许接入同一套 closer。
- 真实拖拽必须优先于 title、文本编辑、option field 打开；非拖拽单击则必须保持原行为。

### 3. 最终验收口径

完成后，用户在主表中应获得以下体验：

- 鼠标拖拽经过多个可见数据格后，矩形选区保留且可见。
- 按 `Delete` 能清空 `Text`、`Number`、`Checkbox`、`Select`、`Multi-select`。
- 若 `Select` / `Multi-select` 的弹层已打开，会先关闭再执行删除。
- 单击 title cell 时，单格选区仍可见，detail panel 同时正常打开。
- 拖拽经过 title / text / option field 时，不会误打开 detail、误进入文本编辑或误开弹层。
