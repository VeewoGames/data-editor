# 视图图标菜单 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 为共享视图增加独立的“更换图标”能力，同时把 `显示为` 固定为未来视图类型切换的禁用预留入口，并确保图标不进入个人 drafts / save-for-everyone 语义链，且顶层视图与组内视图都能贯通同一套图标存储与渲染链路。

**Architecture:** 视图图标作为 shared view leaf item 的元数据，和 `CollectionView` 内容字段分层存放；为保证顶层视图与组内视图都能承载同样的 `icon` 元数据，计划先统一 shared view leaf shape，再接 `shared-view-normalize`、`view-state`、`shared-view-structure` 的复制与结构变换；`App.tsx` 提供直接写 shared config 的更新入口，`ViewTabs.tsx` 只负责菜单交互和渲染。

**Tech Stack:** TypeScript, React, Node test, Playwright, shared view config helpers

---

## 方案概述

### 1. 总体目标和范围

本计划用于落地 [视图图标菜单设计](C:/Code/data-editor/docs/superpowers/specs/2026-06-22-视图图标菜单设计.md)，核心目标如下：

- 视图菜单新增 `更换图标` 能力。
- `显示为` 保持禁用，占位为未来视图类型切换入口。
- 图标状态进入 shared view leaf item 元数据，不并入 `CollectionViewDraft`。
- 图标修改直接写 shared views config，不依赖 `为所有人保存`、`重置`、`viewDrafts`。
- 复制视图、复制视图组、组内结构变换都不丢失图标。

本次范围包含：

- `SharedViewsConfig` / normalize / structure helper / `ViewTabs` / `App.tsx` / 定向测试与 e2e

本次不包含：

- 真实 `viewType` 切换
- emoji / 自定义图片 / 图标颜色
- 组级图标

### 2. 各阶段任务概要

阶段零：图标候选集与 registry 打底。  
主要工作是从当前 `src/components/icons.ts` 图标库中随机抽取一组可替换图标，并把这组候选固化为视图图标 registry；预期成果是菜单候选、类型定义、渲染映射共用同一真相；执行顺序先抽样，再补 registry 测试。

阶段一：shared view leaf shape 统一与 normalize 打底。  
主要工作是把顶层视图和组内视图统一到都能承载 `icon` 元数据的 leaf shape，并补合法值校验；预期成果是旧配置零迁移兼容，非法 icon 自动回退，组内图标链路有真实宿主；执行顺序先红测再改 schema/normalize。

阶段二：`view-state` 与 `shared-view-structure` 保持图标不丢失。  
主要工作是让 duplicate、组复制、结构变换都保留 `icon`；预期成果是图标成为 shared view 身份的一部分；执行顺序先补状态层测试，再改 helper。

阶段三：`App.tsx` 直写 shared config 入口。  
主要工作是新增图标更新动作，并明确不走 `viewDrafts` / `saveSharedViewDraftsToConfig()`；预期成果是菜单点击后立即持久化到 shared config；执行顺序先测行为边界，再补入口。

阶段四：`ViewTabs` 菜单与图标渲染。  
主要工作是新增 `更换图标` UI、让 tab / drag ghost 读取 resolved icon；预期成果是用户可见的图标切换体验闭环；执行顺序先源码结构断言，再接交互。

阶段五：端到端验证与收尾。  
主要工作是补 Playwright 主流程和组场景；预期成果是刷新、复制、组内结构变化后图标仍正确；执行顺序先 top-level，再 group path。

### 3. 整体结构框架

整体结构分三层：

1. 存储层：`.data-editor/shared-views.json` 中的 `kind: "view"` leaf item 持有 `icon` 元数据。
2. 状态层：`shared-view-normalize`、`view-state`、`shared-view-structure` 负责 normalize、复制和结构变换，不让图标进入 draft 语义链。
3. 渲染层：`App.tsx` 暴露图标更新动作，`ViewTabs.tsx` 负责菜单和 tab 图标渲染。

核心边界如下：

- `icon` 不是 `CollectionView` 内容字段。
- `icon` 不进入 `normalizeCollectionViewDraft()`。
- 图标修改不依赖 `saveSharedViewDraftsToConfig()`。
- `显示为` 与 `更换图标` 不耦合。

```ts
export const sharedViewIconIds = new Set(["borderAll", "json", "edit", "tagsField", "refresh", "settings", "filter"]);
const defaultSharedViewIconId = "borderAll";

function normalizeSharedViewIcon(value) {
  const icon = normalizeString(value);
  return sharedViewIconIds.has(icon) ? icon : defaultSharedViewIconId;
}

function normalizeSharedViewLeaf(value, usedViewIds) {
  if (!value || typeof value !== "object" || Array.isArray(value)) return null;
  const rawView = value.kind === "view" ? value.view : value;
  const view = normalizeCollectionView(rawView);
  if (!keepNormalizedView(view, usedViewIds)) return null;
  return {
    kind: "view",
    icon: normalizeSharedViewIcon(value.icon),
    view,
  };
}

function normalizeSharedViewItem(value, usedGroupIds, usedViewIds) {
  if (!value || typeof value !== "object" || Array.isArray(value)) return null;
  if (value.kind === "group") {
    const id = normalizeString(value.id);
    const name = normalizeString(value.name);
    const views = Array.isArray(value.views)
      ? value.views.map((leaf) => normalizeSharedViewLeaf(leaf, usedViewIds)).filter(Boolean)
      : [];
    if (!id || usedGroupIds.has(id) || !name || views.length === 0) return null;
    return {
      kind: "group",
      id,
      name,
      views,
    };
  }
  return normalizeSharedViewLeaf(value, usedViewIds);
}
```

```js
function cloneLeafItem(item) {
  return item.kind === "view"
    ? { kind: "view", icon: normalizeSharedViewIcon(item.icon), view: normalizeCollectionView(item.view) }
    : { kind: "group", id: item.id, name: item.name, views: item.views.map((leaf) => cloneLeafItem(leaf)) };
}

export function createSharedViewConfig(sharedViewsConfig, collectionKey, activeViewId, activeViewSnapshot, options = {}) {
  const config = cloneSharedViewsConfig(sharedViewsConfig);
  const collection = ensureSharedCollection(config, collectionKey);
  const sourceLeaf = findLeafItemByViewId(collection.items, activeViewId);
  const nextLeaf = {
    kind: "view",
    icon: sourceLeaf?.icon ?? "borderAll",
    view: {
      ...normalizeCollectionView(activeViewSnapshot ?? sourceLeaf?.view ?? defaultAllView()),
      id: uniqueViewId(listCollectionViews(collection), activeViewId || "view"),
    },
  };
  collection.items = insertLeafAfter(collection.items, activeViewId, nextLeaf);
  return { config, view: nextLeaf.view };
}

export function createViewInGroupConfig({ sharedViewsConfig, collectionKey, groupId, activeViewSnapshot }) {
  const config = cloneSharedViewsConfig(sharedViewsConfig);
  const collection = ensureCollection(config, collectionKey);
  const nextLeaf = {
    kind: "view",
    icon: "borderAll",
    view: {
      ...normalizeCollectionView(activeViewSnapshot ?? defaultAllView()),
      id: uniqueViewId(flattenItems(collection.items).map((leaf) => leaf.view), "view"),
    },
  };
  collection.items = appendLeafToGroup(collection.items, groupId, nextLeaf);
  return { config, view: nextLeaf.view };
}
```
