# Data Editor Plans

## 概述

本目录只存放未来计划、执行方案、迁移方案和阶段性设计，不承担正式产品文档职责。

正式产品文档请查看上一级 `docs/` 目录。

## 使用规则

- 当前已实现、已生效、已验证的事实：写回 `docs/`
- 未来要做、待决策、待拆解、待执行的内容：写到 `docs/plans/`

## 建议命名

建议使用以下命名格式：

```text
YYYY-MM-DD-主题.md
```

例如：

```text
2026-06-03-relation-batch-maintenance.md
```

## 建议结构

计划文档建议至少包含：

1. 背景
2. 目标
3. 范围
4. 方案
5. 风险
6. 验证方式

如果计划执行后沉淀为正式机制，应把最终结论回写到 `docs/`，而不是长期只保留在 `plans/`。
