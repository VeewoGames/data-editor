# Notion式页签浮框 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 把共享视图页签与视图组菜单从旧的动作菜单式结构重构为 Notion 风格的 header 式浮框，并在不改变现有 shared view icon 存储主链路的前提下，补齐独立 icon picker、搜索、分组、最近使用与对应验证。

**Architecture:** 保持 shared view icon 的持久化链路继续由 `App.tsx` + shared views config 承载，重构重点集中在 `ViewTabs` 交互层。主浮框统一拆成“header 属性编辑区 + 动作区”，header 上的 icon trigger 再挂载一个独立次级 `Popover` 作为 icon picker；图标白名单、二字分组、搜索别名和最近使用规则集中收敛为一组静态元数据，避免散落在菜单实现里。现有 `tests/view-state.test.mjs` 与 `tests/data-editor.spec.ts` 已经对旧菜单结构有硬编码断言，因此实施必须把“旧测试迁移”当成正式任务的一部分，而不是在最后顺手修。

**Tech Stack:** TypeScript, React, Radix Popover, Tabler Icons, Node test, Playwright

## 方案概述

### 1. 总体目标和范围

本计划用于落地 [Notion式页签浮框修订方案](C:/Code/data-editor/docs/plans/2026-06-22-Notion式页签浮框修订方案.md)，并取代旧的 [视图图标菜单实施计划](C:/Code/data-editor/docs/plans/2026-06-22-视图图标菜单实施计划.md)。

核心目标如下：

- 视图菜单与视图组菜单统一成 header 式结构。
- header 顶部直接展示“当前图标 + 名称输入框”。
- 点击 header 图标后，打开独立的 icon picker 浮框，而不是继续把 icon grid 内嵌在主菜单正文里。
- icon picker 第一版就包含搜索框、默认图标区、最近使用区、二字标签页分组和空态。
- `borderAll` 保持默认图标，不被 filled 图标池替换。
- 移除旧菜单里的 `重命名` / `重命名组`、内嵌 icon grid、`来源 / 团队共享视图`。

本次范围包含：

- `ViewTabs` 浮框结构与状态重构
- icon picker 图标白名单、分组、搜索别名、最近使用
- 样式与交互焦点修订
- 现有 Node test / Playwright 用例改造

本次不包含：

- `显示为` 的真实视图类型切换
- 组级 icon 持久化
- emoji、自定义上传图片、远程图标资源、颜色系统

### 2. 各阶段任务概要

阶段零：旧测试迁移与单一真相约束收敛。  
主要工作是先识别并迁移现有依赖旧菜单结构的源码断言和 e2e 选择器，同时固定 `SharedViewIconId` 与 registry keys 必须对齐；预期成果是后续重构不会先被旧测试反向卡死；执行顺序先改测试锚点，再进入 UI 重构。

阶段一：图标元数据与 picker 数据模型收敛。  
主要工作是把 filled 图标白名单、二字标签页分组、搜索别名、最近使用存储 key 固化到统一元数据，并明确本地存储容错与去重规则；预期成果是 icon picker 不再依赖 `Object.keys(sharedViewIconRegistry)` 直接平铺；执行顺序先收敛 registry，再补搜索与最近使用辅助函数。

阶段二：主浮框结构重构。  
主要工作是把视图菜单和视图组菜单都改成 header 式结构，删除旧的重命名模式和无用菜单块；预期成果是打开浮框后直接进入“属性编辑上下文”；执行顺序先做视图菜单，再统一视图组 header 壳子。

阶段三：次级 icon picker 与焦点链路。  
主要工作是实现嵌套 `Popover`、搜索、默认区、最近区、标签页区、全局搜索和空态，并显式锁定“点击 icon trigger / picker 内容时主浮框不关闭”；预期成果是 Notion 式图标选择链路真正可用；执行顺序先跑通图标选择，再补 `Escape` / `blur` / outside click 细节。

阶段四：样式与回归验证。  
主要工作是补 header、picker、分组标签、禁用态、空态样式，并更新现有单测与 Playwright；预期成果是结构与真实交互都有保护；执行顺序先源码断言，再端到端回归。

### 3. 整体结构框架

整体结构分三层：

1. 主触发层：active 页签或 active 视图组二次点击打开主浮框。
2. 主浮框层：header 属性编辑区 + 动作区。
3. 次级浮框层：header icon trigger 挂出的 icon picker。

核心边界如下：

- 视图 icon 仍写 shared view config，不进入 draft 语义。
- 视图组 header 只统一结构和命名输入框，组 icon 按钮第一版禁用。
- 搜索始终全局覆盖白名单，不受当前标签页分组限制。
- `borderAll` 作为默认项单独展示，可以进入最近，但不归入 filled 图标分类池。
- 菜单打开语义保持现状：必须先处于 active，再次点击当前 tab 或当前 group 才打开对应主浮框。
- rename draft 生命周期必须固定：菜单打开时从持久化名称灌入，切换目标时重置，`Enter` / `blur` 提交，`Escape` 回退，关闭菜单时清空。
