# Data Editor 快速开始

## 概述

本文档说明如何启动、停止、验证 `data-editor`，以及如何使用本机项目 registry 在 UI 中切换多个本地项目。

## 环境要求

- Node.js：`>=20 <25`
- 运行目录：仓库根目录 `C:\Code\data-editor`

## 常用命令

```powershell
npm run dev
npm run open -- --project <project-root>
npm run stop
npm run build
npm test
npm run test:e2e
```

## 推荐启动方式

日常使用优先走：

```powershell
npm run open -- --project <project-root>
```

这条命令会：

1. 规范化项目上下文。
2. 确保 recovery bridge 已启动。
3. 通过 controller 启动主服务。
4. 将命令行项目注册到本机 registry。
5. 输出可访问地址。

默认情况下：

- 主服务端口：`8787`
- recovery bridge 端口：`8791`

本机 registry 默认路径：

```text
%APPDATA%\data-editor\projects.json
```

可以用环境变量或命令行参数覆盖：

```powershell
$env:DATA_EDITOR_HOME = "C:\Users\lans\.data-editor"
npm run open -- --project C:\Code\Nocturnel

npm run open -- --project C:\Code\Nocturnel --registry-home C:\Users\lans\.data-editor
```

## UI 中切换项目

打开 `http://127.0.0.1:8787/` 后，左侧 Sidebar 顶部会显示项目选择器：

- 下拉框用于切换 active project。
- 设置按钮用于新增项目、修改项目名称、项目根目录和数据源。
- 切换项目时，前端会重新加载该项目的 files、view config 和 view profiles。
- 项目级配置和备份仍写入对应项目的 `.data-editor/` 目录。

## 自定义数据源

项目设置中的数据源使用一行一个定义：

```text
<id>|<label>|<kind>|<path>
```

示例：

```text
data|Data|relative|data
balance|Balance|relative|configs/balance
shared|Shared|absolute|D:\GameData\shared
```

其中 `kind` 支持：

- `relative`：`path` 相对项目根目录解析。
- `absolute`：`path` 作为绝对路径解析。

文件 API 使用虚拟路径，例如 `balance/items.json`。后端不会直接信任请求路径，而是根据 `projectId` 和 `dataSourceId` 重新解析到已注册的数据源根目录。

## 开发模式和静态模式

- `npm run dev`
  - 面向前端开发。
  - 使用开发服务器。
- `npm run open -- --project <project-root>`
  - 面向正式打开工具。
  - 如果 `dist/` 存在，优先走静态模式。
- `npm run build`
  - 生成静态构建。
- `npm run stop`
  - 停止主服务，并在适用时停止 recovery bridge。

## 健康检查

可用的健康端点：

- `GET /api/health`
- `GET http://127.0.0.1:8791/health`

前者表示主服务存活，后者表示 recovery bridge 存活。

## 任务收尾与服务可用性检查

当开发或验证过程中启动过本地服务、临时端口、Browser、Playwright，或执行过 `stop` / `shutdown` / 进程清理时，收尾前运行：

```powershell
npm run service:finalize
```

该命令会检查默认主服务 `http://127.0.0.1:8787/` 和 recovery bridge `http://127.0.0.1:8791/`，在 bridge 可用且主服务不可用时尝试恢复主服务，并清理明确属于 data-editor 临时 stop 流程的残留资源。

只查看状态、不执行清理或恢复时使用：

```powershell
npm run service:status
```

最终可用入口应始终是：

```text
http://127.0.0.1:8787/
```

## 常见运行事实

- `server.mjs` 负责本地 API 和静态资源服务。
- `open.mjs` 是推荐入口，不只是“打开浏览器”。
- `stop.mjs` 会按 registry home 下的运行时状态尝试安全停止进程。
- recovery bridge 负责主服务重启与恢复，不是前端的一部分。
