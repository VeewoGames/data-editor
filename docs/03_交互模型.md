# Data Editor 交互模型

## 方案概述

本文描述 `data-editor` 的目标交互体验。整体参考 Notion database 的信息组织方式，但不是复制 Notion 的所有功能。核心界面由左侧文件/collection 导航、中间 database 主表、右侧详情面板组成。

## 页面结构

```text
Data Editor
├─ 左侧栏
│  ├─ 文件列表
│  └─ 当前文件的 collection 列表
├─ 顶部工具栏
│  ├─ 搜索
│  ├─ 保存
│  ├─ Reset view
│  ├─ 脏状态
│  └─ 行数摘要
├─ 主表
│  ├─ 字段标题
│  ├─ 字段类型菜单
│  ├─ 单元格
│  └─ 嵌套字段摘要
└─ 详情面板
   ├─ record 标题
   ├─ 顶层字段
   ├─ 嵌套 object editor
   ├─ 嵌套 array editor
   └─ JSON fallback editor
```

## 左侧栏

- 显示 `/api/files` 返回的文件。
- 点击文件后加载 document。
- 如果 document 有多个 collection，显示 collection 列表。
- root array 文件显示 `$` collection。
- root object 文件显示内部 array 字段，例如 `skills`、`traits`。

## 主表

### 默认列

主表默认显示：

- primitive 字段：`string`、`number`、`boolean`、`null`
- primitive array 字段：`string[]`、`number[]`

主表默认不显示为普通列：

- object
- object array
- nested array
- mixed array

这些字段在主表中可以显示为摘要入口，例如：

- `Object(4)`
- `Array(3)`
- `Mixed Array(5)`

### 字段类型

字段显示类型包括：

- `Text`
- `Number`
- `Checkbox`
- `Select`
- `Multi-select`
- `Date`
- `JSON`
- `Nested`

字段类型只影响显示和编辑控件，不自动转换源数据。

### 不兼容状态

当字段值无法用当前显示类型表达时，单元格显示：

```text
不兼容
```

用户可以：

- 切换字段显示类型
- 打开详情面板查看原始值
- 在 JSON fallback 中编辑复杂值

## 详情面板

详情面板用于处理不适合主表直接编辑的数据。

### record 标题

标题优先使用以下字段：

```text
name
name_en
rune_name
skill_name
enemy_name
trait_name
affix_name
keyword_id
id
```

### 嵌套编辑规则

- primitive array：chip list editor
- object：key/value property group
- object array：mini table
- mixed array：JSON textarea fallback
- deep object：可折叠 section 或 JSON fallback

## 保存交互

- 修改后显示 dirty 状态。
- 点击保存前由 server 创建备份。
- 保存成功后清除 dirty 状态。
- 保存失败时保留 dirty 状态并显示错误。
- 保存不允许写入 allowlist 外文件。

## 键盘与可访问性目标

MVP 至少支持：

- `Tab` 在主要控件间移动
- `Enter` 进入单元格编辑
- `Esc` 取消 popover/dialog
- Radix UI 组件保留基础 keyboard/focus 行为

后续可扩展：

- 方向键单元格导航
- 批量选择
- 复制粘贴
