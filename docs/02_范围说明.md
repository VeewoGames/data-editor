# Data Editor 范围说明

## 方案概述

本文定义 `data-editor` 的 MVP 范围、后续扩展范围和明确排除项。当前阶段只做产品定义，不进入实现。

## MVP 范围

### 文件支持

- 读取 `.json`
- 保存 `.json`
- 读取 `.csv`
- 保存 `.csv`
- 默认扫描项目 `data/` 目录下允许的 `.json` / `.csv` 文件

### JSON 结构支持

- root array：
  - 直接作为 `$` collection 显示
- root object：
  - object 内部的 array 字段作为 collection 显示
  - object 内部非 collection 字段作为 root metadata 展示，不在 MVP 中深度编辑
- record：
  - 顶层 primitive 字段进入主表
  - 顶层 primitive array 字段进入主表并以 chips 展示
  - 顶层 object / object array / nested array 进入详情面板

### 主表能力

- 搜索
- 排序
- 隐藏列
- 调整列宽
- 行选择
- 单元格编辑
- 字段显示类型切换
- `不兼容` 状态提示
- 视图配置重置

### 详情面板能力

- 展示完整 record
- 编辑 primitive 字段
- 编辑 primitive array
- 展示和编辑 object
- 展示和编辑 object array
- 对复杂混合结构提供 JSON textarea fallback
- JSON fallback 必须做 parse 校验

### 保存能力

- 保存前创建 `.bak` 备份
- 保存 JSON 时保持 root shape
- 保存 CSV 时保持表格语义
- 保存 API 使用 allowlist，只允许写入 `/api/files` 返回的 `.json` / `.csv`

## MVP 暂不包含

- 完整公式系统
- 多视图类型，例如 board、calendar、gallery
- 多人协作
- Notion 远端同步
- Godot editor plugin 集成
- 完整大文件虚拟滚动
- Web Worker 分片解析
- 字段 schema 持久化到项目配置文件
- 关系字段或跨文件 relation 编辑
- 数据校验规则编辑器

## 后续扩展候选

- 使用 Web Worker 解析大文件
- 虚拟滚动和 pinned columns
- 字段 schema 配置文件
- 文件间 relation，例如 `skills.skill_id` 被 `enemies.skills[]` 引用
- 数据质量检查面板
- diff preview / save preview
- 批量编辑
- schema-driven forms
- board/gallery/calendar 视图
- 导出 filtered CSV/JSON

## 需要进一步确认的问题

- 是否需要把字段显示类型配置写入项目文件，还是仅保存在 browser localStorage？
- 是否需要支持打开 `data/` 以外的文件？
- CSV 是否需要支持自定义 delimiter？
- MVP 是否需要新增/删除行，还是先只支持编辑已有行？
- root metadata 是否需要编辑？
